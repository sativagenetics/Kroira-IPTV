KROIRA IPTV Support

KROIRA IPTV is a media player and source manager for user-provided IPTV sources. It does not provide channels, streams, playlists, subscriptions, media content, accounts, or credentials.

Need help?

Use the GitHub issue templates for reproducible bugs and feature requests. For private or sensitive support matters, email batuhandemirbilek7@gmail.com.

Common issues

  • Source authentication failure: verify the provider URL, username, password, account status, and portal/MAC details with the source provider. KROIRA cannot recover or issue provider credentials.
  • No channels: resync the source, confirm the playlist or provider account has active live/VOD/series entries, and review source diagnostics for invalid, duplicate, or filtered entries.
  • No EPG: confirm the source advertises an XMLTV URL or configure a manual EPG URL, then run an EPG sync from Sources or Guide.
  • Wrong EPG: open manual EPG matching from Guide or source diagnostics, search the XMLTV channel, set or clear an override, and resync guide data.
  • Stream does not play: verify the stream URL works for the account, check whether the source requires authentication, proxy, or companion routing, and run a limited source probe from diagnostics.
  • Store/MSIX install: install from Microsoft Store or a trusted signed package, then restart Windows if the Windows App SDK runtime was updated.
  • Reset app data: use Windows Settings > Apps > Installed apps > KROIRA IPTV > Advanced options > Reset. This removes local sources, credentials, guide data, favorites, and preferences.
  • Export diagnostics: use the source diagnostics export action. Exports are sanitized, but review files before sharing because provider names and non-secret configuration may remain.

Include with your report

If possible, mention the source type (M3U, Xtream, or Stalker), the affected area (Live TV, movies, series, guide data, playback, install, or settings), the app version, Windows version, and the latest sync or health status shown in Sources.

Do not include

Do not send passwords, playlist URLs, MAC addresses, or other private provider credentials in public bug reports.

Content and access disclaimer

KROIRA does not bypass DRM, paywalls, authentication, or access controls. Users are responsible for adding only authorized sources and for complying with applicable provider terms and laws.
